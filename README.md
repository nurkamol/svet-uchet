# Свет·Учёт

Анализ выгрузок умного электросчётчика (EX518, EX18 и совместимые — одно- и трёхфазные)
прямо в браузере: расход по месяцам, оплата по блочному тарифу, графики, прогноз,
предупреждения. Русский, узбекский (латиница и кириллица).
Один статический файл, никакого сервера — данные пользователя не покидают браузер.

**Демо:** откройте `index.html` и нажмите «Демо-данные», либо загрузите `samples/demo.csv`.

## Возможности

- Загрузка .xlsx / .xls / .csv (drag&drop); если колонки не опознаны — их указывает
  пользователь, так подходит выгрузка любого счётчика
- Нет выгрузки — можно ввести показания по месяцам вручную
- Сравнение двух выгрузок: год к году и по порядку (до/после кондиционера)
- Три языка: RU / O'z / Ўз, язык сохраняется и передаётся ссылкой (`?lang=uz`)
- Светлая и тёмная тема (следует системной, пока не выберете сами)
- Работает офлайн и ставится на телефон (PWA)
- Ссылка-шэринг: результат уезжает в самой ссылке, без сервера
- Помесячная таблица с разбивкой по блокам тарифа и суммой в сумах
- Графики: месяцы (кВт·ч + оплата) и суточный расход с подсветкой аномалий
- Перевод статусов счётчика («Обратный порядок фаз» и др.) с пояснениями
- Прогноз текущего месяца, предупреждение о границе следующего блока
- Редактируемые тарифы (localStorage), калькулятор по двум показаниям
- Экспорт CSV, печать/PDF, адаптив, демо-режим

## Деплой на Cloudflare Pages

1. dash.cloudflare.com → Workers & Pages → Create → Pages → **Upload assets**
2. Перетащите папку репозитория → Deploy → сайт поднимется на `имя.pages.dev`

Или через Git: Connect to Git → build command пустой, output directory `/`.
Каждый push обновляет сайт автоматически.

### Домены

Боевой адрес — **`svet.nurkamol.com`**, он же прописан в `canonical`, `og:url` и `sitemap.xml`.
Вторым доменом подключается **`ex518.nurkamol.com`**: ловит тех, кто ищет по модели счётчика.

Подключить (для каждого домена): проект → Custom domains → Set up a domain → ввести имя.
Если `nurkamol.com` уже на DNS Cloudflare, запись создастся сама; сертификат выпустится
за несколько минут.

Дубля в выдаче не будет: `canonical` в `index.html` абсолютный и указывает на
`svet.nurkamol.com` с любого хоста — Google склеит алиас с основным адресом сам.
Выданный Cloudflare `имя.pages.dev` продолжит работать и никуда не денется — по той же
причине он безопасен.

**Смена домена — дорогая операция, не косметическая.** Адрес попадает в ссылки-шэринги
(они живут в чужих чатах вечно) и в origin установленного PWA. Если всё же меняете,
править надо в четырёх местах: `<head>` в `index.html` (canonical, hreflang, og),
константу `SITE_URL` в скрипте, `robots.txt` и `sitemap.xml`.

## og:image

`og-image.png` (1200×630) — превью для репостов в Telegram, Facebook и т. п.
Собран из `tools/og-card.html`; пересобрать после правок бренда:

```bash
python3 -m http.server 8731            # из корня репозитория
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
  --headless --disable-gpu --hide-scrollbars --force-device-scale-factor=1 \
  --window-size=1200,630 --virtual-time-budget=5000 \
  --screenshot=og-image.png http://localhost:8731/tools/og-card.html
```

Через `file://` не сработает: карточке нужны шрифты с CDN.

## Безопасность

Обещание «данные не покидают браузер» подпёрто браузером, а не только текстом:
CSP с `connect-src 'none'` физически запрещает странице отправлять что-либо наружу.
Библиотеки с CDN подписаны через SRI — подменённый файл не выполнится.

**Меняете версию библиотеки — пересчитайте хеш:**

```bash
curl -sL "https://cdn.jsdelivr.net/npm/chart.js@4.4.4/dist/chart.umd.min.js" \
  | openssl dgst -sha384 -binary | openssl base64 -A
```

Готовое значение подставить в `integrity="sha384-…"`. Тест `npm run test:security`
сверяет хеши с настоящим CDN и упадёт, если забыли.

CSP лежит в двух местах — мета-тег в `index.html` (работает на любом хостинге и при
открытии файла напрямую) и `_headers` для Cloudflare Pages (там же то, чего мета не
умеет). При правке менять оба; тест сверяет их между собой.

